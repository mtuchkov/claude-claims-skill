# Commit Message References

Use these as primary references when authoring or validating messages.

## Standards

- Conventional Commits 1.0.0
  - https://www.conventionalcommits.org/en/v1.0.0/
  - Canonical spec for `type(scope)!: summary` and `BREAKING CHANGE`.

## Git Guidance

- `git commit` documentation
  - https://git-scm.com/docs/git-commit
  - Official behavior for commit message handling and hooks.

- `githooks` documentation
  - https://git-scm.com/docs/githooks
  - Official `commit-msg` hook contract and lifecycle.

## Netty-Style Inspiration

- Netty repository commit history
  - https://github.com/netty/netty/commits/4.1
  - Useful examples of concise headers and structured reasoning in bodies.

## Practical Writing Guidance

- Chris Beams: How to Write a Git Commit Message
  - https://cbea.ms/git-commit/
  - Clear, practical guidance on imperative subject lines and readable bodies.
